Где располагается репозиторий с кодом, как посмотреть существующий код;

1: https://github.com/JohnSheff/sf_site

Как устроен процесс внесения своих изменений в основную кодовую базу;

1:Создание новой ветки для задачи(Новых изменений)
2:Выполнение задач
3:Проверки работоспособности проекта(Проверка кода) с выполнеными задачами в ветке(задачи)
4:Слияние с главной веткой

Каковы правила именования веток;

1: Называть ветку именем задачи или номером задания

Как настроить свою среду разработки, требуется прикрепить ссылку на лежащий в проекте конфиг;

1: https://github.com/JohnSheff/sf_site/blob/main/.gitconfig

Инструкция для младших сотрудников по слиянию веток, какие команды надо выполнить, чтобы:

- создать свою ветку;
git branch 'Название ветки'
git checkout 'Название ветки' -переход в ветку

- внести изменения в нее;
git add 'Название файла'
git commit -m 'Название коммита(обычно пишут что было сделано или что было изменено)'
git push origin 'Название ветки'

- слить эти изменения с основной кодовой базой.
git checkout 'Название мастер(Главной) ветки'
git merge 'Название ветки которую хотите объединить с мастер(Главной) веткой'
git branch -d 'Название ветки которую хотите удалить(Если нужно)'
